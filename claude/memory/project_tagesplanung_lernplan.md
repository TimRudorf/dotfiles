**Ziel:** Tim eine professionelle Tages- und Lernplanung aufbauen, damit er nicht in Klausuren-Stress am Semesterende kommt, sondern kontinuierlich übers Semester lernt.

**Phase 1 — Tagesplanung (Kalender, läuft):**
- EDP-Werkstudent: voll flexibel, remote, kein fester Block (siehe `user_arbeit_flexibilitaet.md`)
- TU Darmstadt: feste Vorlesungs-/Übungs-Slots = harte Fixpunkte → Daten kommen aus Moodle
- Rettungsdienst: feste Schichten → Daten kommen separat von Tim
- Sport täglich: Anker, in Tagesplan integrieren
- Mittagessen / Schlaf / Magerquark-Anker: Daten kommen separat von Tim

**Phase 2 — Uni-Lernplan (kommt nach Phase 1):**
- Tim gibt Moodle-Zugriff frei
- Tim teilt mit, welche Klausuren er am Semesterende schreibt
- Lernplan über das ganze Semester verteilt — keine Klausurenphase-Crash-Strategie
- Lernslots werden in die freien Tagesplan-Lücken eingebaut

**Reihenfolge:** Tim hat klar gemacht — Phase 1 erst komplett abgehakt, dann Phase 2.

**Why:** Tim funktioniert über Routinen, ohne Struktur wird er nachlässig (siehe `user_routines.md`). Cut bis 12.06. + Klausuren SS 26 + Werkstudent + Rettungsdienst gleichzeitig — ohne Plan verliert er Bereiche.

**How to apply:** Bei jedem Schritt erst alle nötigen Daten einsammeln, dann als Vorschlag liefern (nicht direkt in den Kalender), erst nach Tims OK persistieren.
