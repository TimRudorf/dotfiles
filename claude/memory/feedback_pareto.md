Tim arbeitet stark nach dem **Pareto-Prinzip**: 80 % Ergebnis für 20 % Aufwand. Gilt nicht nur für Code, sondern auch für Lifestyle-Themen (Ernährung, Routinen, Tools).

**Why:** Tim selbst genannt — er will alltagstaugliche Lösungen, keine perfekten aber schwer durchhaltbaren. Lieber eine simple Regel die hält, als ein perfektes System das nach 2 Wochen kippt.

**How to apply:**
- Bei Plänen / Routinen / Empfehlungen: Standardmäßig die einfachste Variante zuerst vorschlagen, die ~80 % des Ergebnisses bringt.
- Tracking, Logging, Mess-Overhead: minimal halten. Keine täglichen Trackings vorschlagen, wenn 1×/Woche reicht.
- Code & Tools: keine Over-Abstractions, keine "nice-to-have"-Features ungefragt einbauen.
- Nur wenn Tim explizit Tiefgang/Vollständigkeit will, in 100%-Modus wechseln.
